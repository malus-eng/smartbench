# Baseline Measurement: User-space gpiomon + gpioset

## Setup
- HC-SR04 powered from PIN_1 (3.3V)
- TRIG: PIN_13 (gpiochip3 line 2)
- ECHO: PIN_11 (gpiochip3 line 1) — direct connection (3.3V output at 3.3V supply)
- Target: iQOO Neo 7 SE smartphone (164.55 mm length)
- Tool: gpiomon -F "%e %s.%n"
- Trigger: `sudo gpioset gpiochip3 2=1; sudo gpioset gpiochip3 2=0`

## Results

| # | Pulse (us) | Distance (cm) | Notes                |
|---|-----------:|--------------:|----------------------|
| 1 | 923.8      | 15.9          | phone, baseline pose |
| 2 | 3818.1     | 65.8          | distance changed     |
| 3 | 975.7      | 16.8          | phone, baseline pose |
| 4 | 10007.5    | 172.5         | outlier — beam likely missed phone, reflected off wall |
| 5 | 917.9      | 15.8          | phone, baseline pose |

## Observations
- 3 of 5 measurements on the same target cluster within 1 mm (15.8 / 16.8 / 15.8)
- 1 outlier (#4) explained by beam divergence reaching back wall
- Trigger pulse generated by shell-paired gpioset has unpredictable width
  (likely 1-100 ms), far above the 10 us datasheet spec — but the module
  tolerates this thanks to edge-triggered design

## Conclusion
User-space tooling demonstrates that the hardware path works correctly.
The measurement core (ECHO pulse width → distance) is accurate to <5%.
Next: replace ad-hoc shell commands with a kernel driver to enable:
- Deterministic 10 us trigger pulse (via udelay)
- IRQ-based echo capture (eliminates user-space polling jitter)
- Repeated automated sampling (for variance / latency analysis)
